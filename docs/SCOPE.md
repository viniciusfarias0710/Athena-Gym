# SCOPE – Delimitação de Escopo

## Projeto: Site de Academia

---

## 1. Objetivo deste Documento

Este documento define **explicitamente os limites do projeto** e tem como finalidade:

* Evitar expansão indevida de escopo (scope creep)
* Reduzir interpretações ambíguas pelo agente de IA
* Garantir aderência ao MVP definido no PRD

Este documento possui **caráter restritivo** e deve ser respeitado integralmente.

---

## 2. Escopo Permitido (O que PODE ser feito)

O agente de IA está autorizado a:

* Criar site institucional da academia
* Desenvolver páginas estáticas e funcionais do MVP
* Implementar formulários simples de contato
* Integrar botão de redirecionamento para WhatsApp
* Aplicar boas práticas de UX e responsividade
* Utilizar HTML, CSS, JavaScript e frameworks web leves
* Organizar estrutura básica de frontend e backend (quando necessário)

---

## 3. Escopo Proibido (O que NÃO PODE ser feito)

O agente de IA **NÃO** pode, em nenhuma hipótese:

* Criar sistema de autenticação (login/senha)
* Criar área do aluno
* Implementar sistema de pagamento online
* Integrar gateways de pagamento (Pix, cartão, boleto)
* Desenvolver aplicativo mobile
* Criar dashboard administrativo
* Implementar banco de dados complexo
* Criar funcionalidades não descritas no PRD

---

## 4. Funcionalidades Explicitamente Fora do Escopo

As seguintes funcionalidades estão **fora do escopo do projeto atual**:

* Matrícula online
* Controle de frequência de alunos
* Gestão financeira da academia
* Sistema de avaliação física
* Chat em tempo real
* Recomendações de treino

---

## 5. Regras de Conduta do Agente

* O agente deve recusar solicitações fora do escopo
* A recusa deve ser educada, técnica e justificada
* O agente deve referenciar este documento ao recusar
* O agente não deve sugerir alternativas fora do escopo

---

## 6. Critério de Evolução de Escopo

Qualquer alteração neste escopo:

* Deve ser explicitamente solicitada pelo usuário
* Deve gerar uma atualização formal do PRD e/ou SCOPE
* Não pode ser assumida automaticamente pelo agente

---

## 7. Hierarquia de Autoridade

Em caso de conflito:

1. PRD.md
2. SCOPE.md
3. MRP.md
4. system_prompt.md

---

## 8. Encerramento

Este documento é uma **barreira de controle**.
O sucesso do projeto depende da **aderência estrita ao escopo definido**.
